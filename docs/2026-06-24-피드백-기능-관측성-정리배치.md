## 1. P10 개요

P10 = **관측성**(audit 로깅·메트릭) + **정리 배치**(staging·만료 백업) + **문서**. 온프레미스
단일 워커·사내 LAN 전제이므로 Prometheus/OpenTelemetry 같은 외부 의존성을 들이지 않고
**stdlib `logging` + 인프로세스 카운터**로 구현한다(NFR-관측성/저장공간, S6 대응).

### 신규/변경 파일

| 파일 | 내용 |
|---|---|
| `src/web/observability.py` (신규) | `audit(event, **fields)` 구조적 로깅 + 스레드안전 카운터(`incr`/`snapshot`/`reset`) |
| `src/web/maintenance.py` (신규) | `run_cleanup_once`(staging+만료 백업 정리, 순수 함수), 디스크/오디오 저장량 집계 |
| `src/web/store.py` | `prune_expired_backups(max_age)`, `count_backups()`, `count_meetings()` 추가 |
| `src/web/app.py` | lifespan 정리 스케줄러, 핸들러 audit 배선, `require_admin`, `GET /api/admin/metrics` |
| `tests/test_observability_cleanup.py` (신규) | 단위(observability/store/maintenance) + HTTP(메트릭 admin 게이트·카운터) |

## 2. 관측성 (Observability)

### audit 이벤트

`observability.audit(event, **fields)` 는 구조적 한 줄 로그(`event=... k=v ...`)를 남기고 동명
카운터를 +1 한다. **비밀(secret/비밀번호/토큰 값)은 절대 싣지 않는다** — owner(username)·meetingId·
메타만 기록한다. `meetscript.audit` 로거로 흘러 uvicorn stdout 에 보이며, `WEB_AUDIT_LOG` 지정 시
파일에도 남는다.

| 이벤트 | 발생 위치 |
|---|---|
| `meeting.create` / `meeting.delete` | 회의 확정/삭제(오디오 동반 여부 포함) |
| `meeting.conflict_412` / `meeting.patch`(카운터) | PATCH If-Match 충돌 / 성공 |
| `profile.edit` | 표시명 self-edit |
| `regenerate.request` / `regenerate.apply` / `regenerate.undo` / `regenerate.conflict_412` | 재요약 잡 요청·확정·되돌리기·충돌 |
| `audio.staging` / `audio.bind` / `audio.stream`(카운터) | 오디오 업로드·bind·스트리밍 |
| `cleanup.run` / `scheduler.start` | 정리 배치 실행·스케줄러 기동 |

### 메트릭 엔드포인트 — `GET /api/admin/metrics` (admin 전용)

`require_admin`(role=admin) 게이트. 인프로세스 카운터 스냅샷 + 저장 집계를 한 번에 노출(비밀 없음):

```json
{
  "counters": { "meeting.create": 12, "regenerate.apply": 3, "audio.staging_bytes": 10485760, ... },
  "meetings": 12, "backups": 1,
  "audioBytes": 10485760,
  "disk": { "total": ..., "used": ..., "free": ..., "percent": 41.2 },
  "cleanup": { "enabled": true, "intervalSec": 3600, "stagingMaxAgeSec": 86400, "backupMaxAgeSec": 604800 }
}
```

카운터는 단일 워커 프로세스 수명 동안 누적된다(재기동 시 0). 외부 모니터링이 필요하면 이 엔드포인트를
주기 폴링하면 된다.

## 3. 정리 배치 (Cleanup batch)

### 대상과 비대상

- **staging 오디오**: 처리(STT) 시점 업로드됐으나 finalize 안 한 임시 파일 → `cleanup_staging` 으로
  `WEB_STAGING_MAX_AGE_SEC`(기본 1일) 초과분 삭제(FR-C1: 미확정 staging 비영속).
- **재요약 백업**: apply 후 undo 안 한 `meeting_backup` 누적 → `prune_expired_backups` 로
  `WEB_BACKUP_MAX_AGE_SEC`(기본 7일) 초과분 삭제. 만료돼도 회의 본문은 이미 apply 로 반영됨
  (만료 시 undo 여력만 소멸).
- **draft 회의(비대상)**: 이 앱은 finalize 시점에만 회의를 영속한다 → 영속 draft 고아가 없다.
  처리중(processing) 회의 자동삭제는 인플라이트 잡 유실 위험이라 정리 대상에서 제외한다.

### 스케줄

`app.py` lifespan 이 startup 시 `_cleanup_loop` 를 띄운다: 부팅 직후 1회 + 이후
`WEB_CLEANUP_INTERVAL_SEC`(기본 1시간) 주기. 블로킹 디스크 IO 는 `asyncio.to_thread` 로
워커스레드에 위임(이벤트 루프 비차단). 한 사이클 예외는 로깅 후 삼켜 루프가 죽지 않는다.

**스케줄러 가동 조건**: `WEB_CLEANUP_ENABLED != "0"` **그리고** `MEETSCRIPT_BLOCK_DEFAULT_DB != "1"`.
후자는 테스트 가드(conftest) — 테스트에선 스케줄러를 띄우지 않고 `maintenance.run_cleanup_once` 를
직접 호출해 정리 로직을 단언한다.

### 환경 변수

| 변수 | 기본값 | 의미 |
|---|---|---|
| `WEB_CLEANUP_ENABLED` | `1` | 정리 스케줄러 마스터 스위치(`0`=비활성) |
| `WEB_CLEANUP_INTERVAL_SEC` | `3600` | 정리 주기(초) |
| `WEB_STAGING_MAX_AGE_SEC` | `86400` | staging 보존 기간(초, 초과 시 삭제) |
| `WEB_BACKUP_MAX_AGE_SEC` | `604800` | 재요약 백업 보존 기간(초) |
| `WEB_AUDIT_LOG` | (없음) | 지정 시 audit 를 해당 파일에도 기록 |
| `WEB_AUTH_ADMINS` | `admin` | 메트릭 엔드포인트 접근 가능한 admin 사용자(콤마구분) |

## 4. 피드백 이니셔티브 전체 요약 (트랙 A/B/C)

P0~P10 전 단계가 두 repo(LB_Note 백엔드 / meetscript-ai 프론트)에 머지 완료됐다.

| 트랙 | 핵심 | 비고 |
|---|---|---|
| **A 표시명** | profile self-edit + `name_source` 컬럼별 조건부 seed(display 보호, role 유지) | seed 재실행·PRUNE 정합 |
| **B 콘텐츠 교정** | transcript(위치 식별)·summary(item_id·구조보존) 교정 PATCH + If-Match 낙관적 동시성 | edited 서버 set, grounding 우회+근거 실재 |
| **C 오디오/재요약** | 오디오 staging/bind/Range, `<audio>`·점프, 재요약(item_id prompt 미리보기·백업·undo) | Phase I(item_id) 선결, 이중 전송 제거 |

### 운영 주의

- **단일 워커 전제**: in-memory 락(store)·카운터(observability)는 단일 uvicorn 워커에서만 유효.
  멀티 워커로 확장하면 락·카운터·잡 dict(`_jobs`)를 외부화해야 한다.
- **백업/undo 수명**: 재요약 undo 는 `WEB_BACKUP_MAX_AGE_SEC` 내에서만 가능(만료 후 본문은 유지).
- **테스트 격리**: 모든 테스트는 `tempfile` + `DEFAULT_DB_PATH` 패치로 실 DB 미접촉. conftest 가
  `MEETSCRIPT_BLOCK_DEFAULT_DB=1` 로 실 운영 DB(`output/web/meetings.db`) 접촉을 차단한다
  (과거 실 DB 변조 사고 재발 방지).

## 5. Follow-up (P10 비범위)

- 재요약 `preserve_edited`/`overwrite` 모드(item_id 자동병합) — 현재 prompt(전체 교체/취소)만.
- JobError `error_code`(claude_auth_expired)의 STT 경로 UX 전파.
- 오디오 Range 시킹 정밀도(transcript numeric start 도입).
- 운영자 강제 이름 동기화(`WEB_AUTH_FORCE_NAME_SYNC`), 멀티파트 재시도/이어올리기.
- 디스크 임계 알람(현재 메트릭 노출까지 — 임계 초과 알림은 외부 폴링/연계로).
