# 액션아이템 추출 기준 (2026-06-09)

- 관련 Jira: WDLABD2411-531 (LB NOTE Phase 1)
- 자매 문서: `docs/2026-06-09-summarize-stage-design.md`(요약 스테이지), `docs/2026-06-04-postprocess-pipeline-design.md`
- 근거 자산: 프롬프트 `prompts/extract.ko.md`(extract-ko-1.0), 스키마 `src/postprocess/extract_schema.py`,
  정답셋 `eval/gold_actionitems.json`(10건), 골든 픽스처 `tests/fixtures/extracted_axfull.golden.json`(16건),
  결정적 채점 `src/postprocess/score_extraction.py`, 실추출 산출 `output/extract/text-axfull.actionitems.json`(8건)
- 목적: 흩어진 프롬프트·정답셋·채점기에 분산된 추출 기준을 **하나로 명문화**(역분석 + 정답셋 대조).
- 전제: **화자분리(diarization) 불필요** → 실제 회의록은 담당을 **팀/부서**로 적으므로(개인 화자 X)
  `owner` 는 본문에 팀/부서가 명시되면 채우고, 아니면 null(추측 금지).

## 1. 무엇을 액션아이템으로 보나 (포함 기준)

회의에서 **누군가 하기로 한 일 / 결정된 후속조치 / 합의된 과제**. 미래 행동이 명확히 드러나는 발화.
- 작성·테스트·보고·구현·공유·정의·제안·확인 등 **실행 동사 + 대상**이 있는 것.
- 실추출 8건 예시(전부 이 패턴): "모델 성능 테스트하고 비교한다", "결과를 문서로 작성해 슬랙으로
  보고한다", "회의록 양식을 작성해 공유한다", "마일스톤 일정표를 작성해 보고한다".

## 2. 무엇을 빼나 (제외 기준)

- 단순 정보공유·잡담·과거 사실 진술(행동 약속이 아닌 것).
- 회의 자체에 대한 메타 발화("녹음되나요" 등).
- transcript 에 **근거가 없는** 항목(할루시네이션 절대 금지).
- **경계항목 정책**: 정답셋(`gold_actionitems.json`)에서 경계항목 11·12 는 **채점 제외 확정**.
  → "액션처럼 보이지만 합의·실행 약속이 약한 발화"는 넣지 않는다(과추출보다 정밀도 우선).

## 3. 형식 규칙

- **text**: 무엇을 할지 **한 문장 명령형**(간결). 정보(숫자/고유명사/용어)는 보존
  (예: "Qwen 3.0", "Cohere", "슬랙", "2장 내외", "대표님").
- **입도(중요)**: 한 과제 = 한 항목. 흩어진 발화는 **한 항목으로 병합**(`evidence_seg_ids` 합집합).
  실추출 id 0 은 seg 7개(15·66·67·68·80·88·162)를 한 항목으로 병합 — **과도 분해 금지**.

## 4. 필드 규칙 (스키마 = `ActionItem`, 웹 actionItems 계약과 동일)

| 필드 | 규칙 |
|---|---|
| `text` | 명령형 한 문장(§3). |
| `owner` | **팀/부서가 본문에 명시된 경우만** 채움(예 `SW2팀`·`개발본부`·`영업/구매`). 불확실하면 null. **화자분리 불필요** — 실제 회의록은 담당을 개인 화자가 아니라 **팀/부서**로 적기 때문(`samples/EYEL-S3000ABR 데모시연 회의록.pdf` 확인). 추측 금지(1인칭 "제가 하겠다"는 팀 불명이면 null). |
| `due` | 발화에 나온 표현 그대로. 실제 양식 예: `6월 17일`·`5/29`·`최대한 빠르게`·`상시`·`미정`. ISO 날짜 명확하면 함께. 없으면 null. |
| `evidence_seg_ids` | **근거 segment id 목록(필수, ≥1).** 그라운딩·환각 방지 핵심. 본문에 실제 등장한 seg 만 인용. |
| `anchor` | **호출부가 결정적 산출**: `evidence_seg_ids` 의 최소 start → `seconds_to_timestamp(min)` (MM:SS). LLM 출력 anchor 는 신뢰하지 않음. |
| `flag` | 게이트가 채움(`확인필요` \| null). |

## 5. 평가 기준 (정답셋 대조 — 결정적 채점)

`score_extraction.py` 가 **keyword 그룹 매칭**으로 회수율(recall)을 결정적으로 산출한다. LLM-judge 의
비결정성을 피하려는 설계.

- **정답 1건 구조**(`gold_actionitems.json`): `text`, `due`, `keywords`(그룹 목록, **그룹 내 OR**),
  `required`(반드시 충족해야 하는 그룹 = 변별자), `min_match`(충족해야 할 최소 그룹 수).
  - 예 gid 1: `keywords=[["모델"],["확정","완료","페이즈1"]]`, `required=[0]`, `min_match=2`
    → 추출 text 에 "모델"(필수) **그리고** {확정|완료|페이즈1} 중 1개가 있으면 커버.
- **커버 판정**(`_covers`): required 그룹을 **모두** 충족 + 충족 그룹 수 ≥ `min_match`.
- **회수율** = `n_covered / n_gold`. owner/due/anchor 는 **미채점**(구간 힌트만, scoring_note 명시).
- **현재 고정 성적**(`tests/test_score_extraction.py`, 회귀 락):
  - axfull = **10/10**(100% recall) — 메인 음원, 합격.
  - axenh = **7/10**(70%) — 미커버 gid 4(서버 vs 로컬)·6(자동저장)·9(케이스 확장).
  - 방향 불변식: **axfull ≥ axenh** 유지(`test_verdict_direction_holds`).

### 합격선(잠정)
- 메인 음원(axfull) recall **= 1.0 유지**가 회귀 기준선.
- 신규 음원은 recall **≥ 0.8** 착수 기본값, 1차 측정 후 사용자 확정.
- 정밀도(오탐): 경계항목 제외 정책(§2)으로 **과추출 억제**. 실추출 8건은 정답 10건 대비 병합·압축된
  형태(1:1 아님) — 회수율로 보되 명백한 환각(근거 없는 항목)은 0 이어야 함.

## 6. 기준을 만들고/개선하는 절차 ("어떻게 세우나")

결과물·정답셋이 이미 있으므로 **역분석 + 정답대조 루프**로 기준을 세운다:

1. **결과 역분석** — 뽑힌 항목들의 공통 형식·입도·필드 규칙을 명문화(본 문서 §3·§4가 그 산물).
2. **정답 대조** — `score_extraction.py` 로 회수율 측정. 미커버 gid 를 보고 **놓친 유형**을 식별
   (예 axenh 의 gid 4·6·9 = 서버운영/자동저장/케이스확장 누락).
3. **경계 라벨링** — "뽑힐락말락한" 발화를 사람이 포함/제외 판정 → 정답셋의 `required`/`min_match`
   또는 경계 제외(11·12)로 박제.
4. **프롬프트 반영** — `prompts/extract.ko.md`(extract-ko-1.0)만 수정(코드 무변경). 프롬프트 = 버전관리
   자산. 회귀 테스트로 axfull=10/10 불변 확인 후 버전 올림(예 1.1).
5. **반복** — 새 음원/정답이 생길 때마다 1~4 재실행. recall·방향 불변식이 회귀 게이트.

## 7. 실무 정합 — 사람 서기 프로세스와의 매핑

실무(회의 퍼실리테이션·PMO)에서 액션아이템은 LLM 일괄추출이 아니라 **회의 중 캡처 → 말미 복창합의
→ 회의 후 추적**의 3단계로 산출된다. 자동 파이프라인은 그중 "복창합의"가 빠지므로 **그라운딩·정답셋**
으로 신뢰를 대체한다. 본 추출 기준이 따르는 실무 원칙:

- **약속(commitment) ≠ 주제·결정.** 액션은 "누가·무엇을·언제까지 하기로 합의한 실행 단위"다.
  정보공유·논의·결정과 분리한다(결정은 요약 `decisions`, 액션은 본 스테이지). → §1·§2.
- **구성요소(업계 표준):** What(동사로 시작, 완료판정 가능) · Who(단일 책임, 팀이어도 1팀) ·
  When(기한) · 근거(추적성). → `text`/`owner`/`due`/`evidence_seg_ids` 가 1:1 대응. → §3·§4.
- **트리거 신호:** *행동 동사 + (담당) + (기한)* 패턴, "~하기로/~할게요/다음까지 ~". 애매하면
  넣지 않음(Parking Lot = 제외). → §2 경계정책.
- **복창합의(Read-back) 대체:** 사람은 회의 말미에 액션을 읽어 담당·기한을 그 자리서 확정한다.
  자동화는 이 단계가 없으므로 **`evidence_seg_ids` 그라운딩 + `anchor`(원문 시점) + 정답셋 회수율**
  (§5)로 사후 검증 가능성을 확보한다. = 환각 차단의 실질 기제.
- **단일 책임(DRI):** owner 는 팀/부서 1개. 복수·공백은 안티패턴 → 불명확하면 null(추측 금지). → §4.
- **정밀도 우선:** 과추출(없는 약속 생성)보다 누락이 낫다. 경계항목 제외로 오탐 억제. → §2·§5.
- **추적성:** 표준 스키마라 Jira/이슈트래커로 이관 가능 — "추적 안 되는 액션은 무의미"라는 실무 철칙.

## 8. 미해결 / 사용자 확정 필요

- 신규 음원 recall 합격선(≥0.8 잠정) 확정.
- 정밀도(오탐) 정량 지표 도입 여부 — 현재는 회수율 중심, 과추출은 경계정책으로만 억제.
- `owner` 채움: 화자분리 제외 확정이므로 **명시 역할만**. 향후 diarization 도입은 별 과제.
- 프론트 `extract-actions` 계약(string[] vs 리치객체) — `meetscript-frontend-integration` 참조, 별 합의.

## 9. v1.4 고도화 (2026-06-17, 회의 결정으로 갱신)

> 상세 결정·근거·시나리오 진단: **`docs/2026-06-17-action-item-methodology-upgrade.md`**.
> 본 절은 위 §1~§8 중 v1.4 에서 **바뀐 점만** 요약한다(상위 문서가 정본).

- **정의 확장(A)**: §1 의 "산출물이 남는 행동" 한정을 폐기. **협의·검토·조율·재확인·승인요청·회신**처럼
  산출물이 안 남는 행동도, **대상이 명시되고 완료판정이 한 문장으로 떨어지면** 액션. 약동사 단독은 제외.
- **owner 3단계(B)**: §4 의 "명시만/추측 금지"를 **명시(explicit) / 본문 앵커 추론(inferred) / null**
  3단계로 확장. 추론은 본문에 "주제↔팀 결속 발화"가 있을 때만, `owner_source="inferred"` +
  `flag="추정"` 으로 **격리**(owner 본필드 추측 주입 금지). 정적 주제→팀 글로서리 자동매핑 금지.
- **모호 발화(C)**: §2 "정밀도 우선 제외"를 부분 완화. 행동·대상은 분명하나 확정이 약한 회색지대
  (특히 고객·벤더 약속)는 버리지 말고 **`flag="약함확인"`** 캡처. 순수 약동사·조건부·완료·선언은 계속 제외.
- **flag 3종 분리**: `확인필요`(근거0 환각, 호출부가 채움) / `약함확인`(모호 캡처, LLM) / `추정`(추론 owner, LLM).
  `n_flagged` 는 사유별 분리 집계(`extract_handoff.py`). (선결 버그: 구 `extract_handoff` 가 LLM flag 를
  전량 폐기 → v1.4 에서 보존하도록 수정.)
- **정밀도 정량화(D)**: §5 의 회수율 단독을 보완. `eval/gold_actionitems.json` 에 **`negatives[]`**(오탐 라벨,
  positives 와 동형 스키마) 신설, `score_extraction.score_precision()` 가 결정적 산출. 게이트:
  `axfull recall=1.0` 유지 + `confirmed_FP` 회귀 락 + `precision_strict` 락 + **방향 불변식 `axfull≥axenh`**(precision 도).
  동결 골든 baseline: axfull precision_strict=0.8125·confirmed_FP=1(idx12 조건부 누수, v1.4 재추출 목표=0),
  axenh precision_strict=0.8·confirmed_FP=0.
- **코드 무변경 원칙 종료**: §6-4 의 "프롬프트만 수정"은 v1.4 에서 끝난다. B·C·D 는
  `extract_schema.py`(owner_source)·`extract_handoff.py`(flag 보존)·`score_extraction.py`(precision) 동반 변경 필수.
- **일반화(E)**: few-shot·정의를 **비개발·벤더·CS 도메인**까지 확장(개발 편향 해소). 멀티도메인 정답셋
  (`eval/gold/*.json`)·도메인별 합격선은 실제 도메인 transcript 확보 후 확정(후속).
