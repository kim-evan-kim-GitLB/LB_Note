## 2. 최우선 발견 — 단일 버그가 4개 증상을 만든다

> **개발자 리뷰어가 캡처 매니페스트의 초기 가설("파이썬 isoformat 마이크로초 6자리 파싱 실패")을 반증했다.**
> 실제 데모 API `/api/meetings` 응답에는 `date` 필드가 **아예 없고**(`date: null`), 옆에 `createdAt: "2026-07-16T08:25:04.638823+00:00"` 이라는 **정상 파싱되는 값이 존재**한다. 프론트가 존재하지 않는 `meeting.date` 를 읽어 `new Date(undefined)` = Invalid Date 가 전파되는 구조.

이 한 가지가 동시에 유발하는 증상 4개 (8명 중 7명이 각자 재발견):

| # | 증상 | 위치 |
|---|------|------|
| 1 | 전 화면 "Invalid Date" 도배 (대시보드·회의 기록·검토 헤더·모바일) | `Dashboard.tsx:166,325`, `ReviewView.tsx:955` |
| 2 | 통계 모순 — "전체 회의 3" vs "회의 기록 0" (`getMonth()` → NaN → 항상 0) | `App.tsx:227-230` |
| 3 | "최신순/오래된순" 정렬 토글 무동작 (`getTime()` → NaN 비교 무의미) | `App.tsx:210-211` |
| 4 | 검토 헤더 "INVALID DATE • " duration null 꼬리 | `ReviewView.tsx:955` |

**수정**: `new Date(meeting.date ?? meeting.createdAt)` 폴백 (파싱 지점 3곳) — 사실상 한 줄 수준으로 4개 증상 전부 해소. (백엔드가 `date` 를 내려주도록 하는 근본 수정은 별도 검토.)

## 3. 관통 테마 (페르소나 교차 종합)

### T1. 신뢰 붕괴 3종 세트 — 첫 화면에서 다 보인다 (전원 지적)
Invalid Date(§2) + 통계 모순 + **모바일 '새 회의 시작' 버튼 세로 붕괴**("새/회/의/시/작" 한 글자씩). 개발자 리뷰어 표현: "QA 한 번 돌렸으면 로그인 3초 만에 봤을 것들이 정교한 자동저장·낙관적 락 위에 얹혀 있다."

### T2. 모바일 헤더 2곳 붕괴 — 사이드바는 정상 (모바일·개발자)
- 대시보드 헤더: `flex justify-between` + wrap 부재, 제목 `text-4xl` 고정, 버튼에 `whitespace-nowrap`/`shrink-0` 없음 + `scale-110` + 컨테이너 `p-12` → CTA 세로 분해. (`Dashboard.tsx:86,89-101`)
- 검토 헤더: 제목 input `w-96`(384px 고정)이 390px 뷰포트 독식 → **'회의록 확정'/'원본으로 원복' 버튼이 화면 밖** — 폰으로 회의록 확정 자체가 불가. (`ReviewView.tsx:939,949,996,1004`)
- 상태 배지('확정됨·Drive'/'초안'/'자동 저장됨')가 전부 `hidden sm:flex` → 모바일 저장 피드백 0. (`ReviewView.tsx:964-987`)
- ⚠️ 오탐 방지: 사이드바는 이미 오프캔버스 드로어+햄버거로 반응형 정상(`App.tsx:513-521`) — 손대지 말 것.

### T3. 이메일 발송 — 본문 편집·미리보기 불가 (대표·비개발자·헤비·개발자 4명)
발송 모달에 제목/To/Cc만 존재, `onSend(m, {to,cc,subject})` 에 body 파라미터 자체가 없음(`ReviewView.tsx:53,849,1746-1821`). 인사말 한 줄 못 붙이고, 뭐가 나가는지 보지도 못한 채 발송. → 최소: 본문 읽기전용 미리보기. 권장: 머리말(개인 메시지) 입력란 + 본문 편집.

### T4. 외부 연동 버튼 — 상태·이유·안내 3중 불일치 (대표·비개발자·개발자)
- Gmail/Drive: 진짜 `disabled`(서버 미설정)인데 이유 미표시 → "왜 회색인지" 알 수 없음. 클릭 시 "Google 연결하기" 유도로 전환 권장(`ReviewView.tsx:1568`, 연결 배너 로직 1620-1632 재활용).
- Slack/Jira/Notion: 클릭되지만 `notImplemented` 토스트만(`ReviewView.tsx:677,1576-1588`). 그런데 옆 '?' 가이드는 **존재하지 않는 "설정 메뉴"에 Webhook 을 입력하라고 5단계 안내**(`:1861`) — 안내와 실동작 정반대. '준비 중' 배지 명시 + 허위 가이드 제거.

### T5. 저장 개념 4개 공존 — 자동저장·요약 저장·확정·원복 (비개발자·헤비)
'자동 저장됨' 배지가 떠 있는데 '요약 저장' 버튼이 따로 있고 '회의록 확정'·'원본으로 원복'이 나란히(`ReviewView.tsx:979-995,414,347,234`). 게다가 배지는 편집 안 해도 시드 `updatedAt` 만으로 렌더되어 "내가 뭘 저장했지?" 혼란(개발자 #9). → 자동저장으로 통일(수동 저장 제거), '확정'만 명시적 액션 + 결과 설명 한 줄("확정하면 Drive 보관·수정 잠금"), 배지는 실제 자동저장 발생 세션에서만.

### T6. "여러 개를 한꺼번에"가 통째로 부재 (헤비유저 관통 진단)
편집 단위 기능(인라인 수정·재요약 undo·낙관적 락)은 정교한데 일괄 처리가 전무 — 다중 파일 업로드 불가(`FileUploadView.tsx:33,87-95` 단일 File), 일괄 삭제/확정 없음(매건 확인 다이얼로그), 액션 통합 뷰는 상태 드롭다운 하나뿐(`ActionItemsView.tsx:87-94`), 검색은 제목 한정·상태/기간 필터 없음(`Dashboard.tsx:282,292`), 삭제 undo/휴지통 없음. 액션 담당자(owner)는 명부 연동 없이 자유입력(`ReviewView.tsx:1294-1303` — 참석자는 명부 피커 있음, 반쪽 연동).

### T7. 관리자 콘솔 — 비가역 작업 가드 불균형 (IT관리자)
사용자/공지 삭제엔 확인 모달이 있는데:
- **Google OAuth 설정 '삭제'만 확인창 없음** — 오클릭 1회로 전 직원 Google 연동 즉사(`AdminGoogleOAuthConfig.tsx:77`).
- **초기 비밀번호 1회 노출에 복사 버튼 없음**(`AdminUserManagement.tsx:659` — Redirect URI엔 복사 버튼 있어 비일관).
- **공지 배포 채널이 웹 어디에도 안 보임** — `SLACK_NOTICE_CHANNEL` 미설정 시 명령 친 채널로 오발송 가능(`slack_bot/handlers.py:124`). 배포 이력(누가·언제·어디로)도 UI 미노출(audit 로그는 백엔드에 존재, `app.py:1970-2007`).

### T8. 접근성 기초 결함 (접근성·개발자)
- 로그인 입력칸 placeholder만, label/aria 없음(`App.tsx:454-471`).
- 아이콘 버튼(X·?·참석자 삭제) aria-label 없음(`ReviewView.tsx:1748,1849,2183`, `ParticipantsView.tsx:515,567,752`).
- 회의 행·통계 카드가 `<div onClick>` — Tab/Enter 불가(`Dashboard.tsx:154-157,313,369-371`).
- `text-[9px]` + `text-text-secondary/50` 대비 미달(WCAG AA 4.5:1).
- 모달 3개 동작 제각각 — 재요약 모달만 role/aria/Esc/backdrop 완비, 발송·가이드 모달은 X 버튼만(`ReviewView.tsx:1654-1657` vs `1735-1748,1836-1849`).
- 잘한 점(유지): 포커스 링 일관, 상태 배지 색+아이콘+텍스트, 기본 16px, 시맨틱 헤딩.

### T9. 온보딩·용어 부재 (신입·비개발자)
첫 로그인 후 "뭐부터?"를 짚어주는 신호 없음. '회의별 액션' vs '텍스트 액션 추출' vs '지능형 액션 아이템' 이름 3종. '보정'이 무엇인지 설명 없는 녹음 시작 버튼 2개(`RecorderView.tsx:529,577`). 'LOCAL AI INFERENCE ENGINE'·모델명 등 기술 용어 노출. 'AI 분석 요약' 카드는 "추후 제공 예정" 영구 플레이스홀더(`Dashboard.tsx:249-258`) — 대표 페르소나: "요약이 아니라 변명."
